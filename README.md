# hackathon アイデア

## 課題（やること）

このサイクルでAI（自分と似た存在）を最適化していく

第１段階　モチベUP

→楽しく(AIがToDo(スケジュール)、本、楽曲を提案)

第ｎ段階　習慣的に記録

→何かが育つ（ＵＩの変化（育成・コレクション要素））

→木などの植物（建物）の成長、バッチ（アチーブ）

最終段階　向上心UP

→習慣的に記録することでAIが分析（スコアリング）

→より良い自分になるために挑戦

---

## 機能要件

todo管理は二の次で（ベースはToDo登録）、一日のモチベーションをあげることがメイン

AIがTodoを提案する機能もあり

習慣的に記録することでAIが分析

カレンダーアプリを連携してデータを取得→AI分析からの提案

連続記録で成長する（木や建物）

記録が途切れたら、枯れるまでのカウントダウンがはじまって

その間に記録したら、もとの成長フェーズにもどる

カウントダウン切ったら、なくなる

### コア部分

※AIからの提案機能よりもまずグラフィカルな変化


### UI変化（トップ画面）の構想

アイソメトリックなデザインを採用

→2D画像を扱いながらも3Dを表現可能

→2次元配列position[i][j]に対して座標(x,y)をx=j-i,y=j+iで変換

それぞれのブロックの中心に木や草、建物が増えていくイメージ

アチーブ（コレクション）の要素として、特別な建物がゲットできる

建物の種類はユーザーのレベルによって増える

ToDoの達成度合い（これを経験値とする）によって建物のグレードが上がる

UIの種類は以下のような感じ

レベルn（右側は高レベルに取得可能）（レベルが3上がるたびに新パーツ解除）（地形マスは10おきに獲得可能）（それぞれ3種類）

「木」、「建物」、「動物」 | 「地形（複数マス）」

レベルはマイクラの仕様を利用

（レベル1-2は9、レベル2-3は11のように2ずつ必要経験値が増えてく）

画面遷移時に生えている要素がアニメーション付きで生成されるようなUI変化を加えたい

また、背景を単調な背景にするよりも風が吹いている様子だったり、雲が流れている背景が望ましい
